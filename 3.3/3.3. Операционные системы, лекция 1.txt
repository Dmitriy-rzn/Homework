7)Чем отличается последовательность команд через ; и через && в bash? 
через ; команды выполняются последовательно не зависимо от результата выполнения предыдущей, && 
команда 2 будет выполнена только в том случае, если команда команда 1 была выполнена успешно.  
Есть ли смысл использовать в bash &&, если применить set -e?
Эта опция приведет к немедленному завершению работы сценария bash в случае сбоя команды 
8)Из каких опций состоит режим bash set -euxo pipefail и почему его хорошо было бы использовать в сценариях?
set -e указывает bash немедленно выйти, если какая-либо команда [1] имеет ненулевой статус выхода
set -u Если установлено, ссылка на любую переменную, которую вы ранее не определили, за исключением $ * и $ @,это ошибка и вызывает немедленный выход из программы
set -o Этот параметр предотвращает маскировку ошибок в конвейере
set -x Эта опция заставляет bash печатать каждую команду перед ее выполнением
помогает в отладке работы сценариев
